# Projekt-Regeln

## Git

- **Keine Git-Aktionen ohne explizite Aufforderung.** Das schließt ein: `git add`, `git commit`, `git push`, `git checkout`, `git branch`, `git reset`, `git rebase`, `git stash`, PR-Erstellung via `gh`, Merge-Operationen — wirklich alle schreibenden Git-Operationen.
- Lesende Git-Befehle (`git status`, `git log`, `git diff`) sind ok, um Kontext zu gewinnen.
- Wenn Git-Aktionen nötig erscheinen, vorher explizit fragen.
